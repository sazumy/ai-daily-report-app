# テスト仕様書

## テスト方針

| 項目 | 内容 |
|------|------|
| テスト種別 | APIテスト（エンドポイント単位）、画面テスト（シナリオ単位） |
| テストID体系 | `API-{エンドポイント番号}-{連番}`、`SCR-{画面番号}-{連番}` |
| 合否基準 | 期待結果と完全一致で合格（PASS）、不一致で不合格（FAIL） |

### テストデータ前提

| データ | 内容 |
|--------|------|
| 営業担当者 | name: 山田 太郎、email: taro@example.com、role: salesperson |
| 上長 | name: 佐藤 部長、email: sato@example.com、role: manager |
| 顧客A | name: 鈴木 一郎、company_name: 株式会社ABC |
| 顧客B | name: 田中 花子、company_name: 有限会社XYZ |
