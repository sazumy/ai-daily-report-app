# 営業日報システム 要件定義書

## システム概要

営業担当者が日々の訪問活動・課題・翌日計画を記録し、上長がフィードバックできる営業日報システム。

---

## ユーザー・ロール

| ロール | 説明 |
|--------|------|
| 営業担当者 (salesperson) | 日報を作成・編集する |
| 上長 (manager) | 全営業の日報を閲覧し、コメントを追加する |

> 上長と部下の紐付けなし。上長は全員の日報を閲覧できる。

---

## 機能要件

### 1. 日報管理

- 営業担当者は1日1件の日報を作成・編集できる
- 日報に記載する内容:
  - **訪問記録**（複数行）
  - **Problem**（今日の課題・相談）
  - **Plan**（明日やること）
- ステータス管理なし

### 2. 訪問記録

- 1つの日報に複数の訪問記録を追加できる
- 訪問記録の構成:
  - 顧客マスタから顧客を選択
  - 訪問内容（自由記述）

### 3. コメント機能

- 上長は日報全体に対してコメントを追加できる
- 1つの日報に複数コメント可能

### 4. マスタ管理

#### ユーザーマスタ

| カラム | 説明 |
|--------|------|
| name | 氏名 |
| email | メールアドレス |
| role | salesperson / manager |

#### 顧客マスタ

| カラム | 説明 |
|--------|------|
| name | 顧客名 |
| company_name | 会社名 |
| phone | 電話番号 |
| address | 住所 |
