# 画面定義書

## 画面一覧

| # | 画面名 | URL | アクセス権限 |
|---|--------|-----|-------------|
| 1 | ログイン画面 | `/login` | 全員（未ログイン） |
| 2 | 日報一覧画面 | `/reports` | 全員（要ログイン） |
| 3 | 日報作成画面 | `/reports/new` | 営業担当者のみ |
| 4 | 日報詳細画面 | `/reports/:id` | 全員（要ログイン） |
| 5 | 日報編集画面 | `/reports/:id/edit` | 営業担当者（自分の日報のみ） |
| 6 | 顧客一覧画面 | `/customers` | 全員（要ログイン） |
| 7 | 顧客作成画面 | `/customers/new` | 全員（要ログイン） |
| 8 | 顧客編集画面 | `/customers/:id/edit` | 全員（要ログイン） |

> **共通仕様:** 未ログイン状態で要ログインページへアクセスした場合は `/login` へリダイレクト。

---

## 各画面の詳細

---

### 1. ログイン画面

**URL:** `/login`
**アクセス権限:** 全員（未ログイン）

> ログイン済みの場合は `/reports` へリダイレクト。

#### 入力項目

| 項目名 | 種別 | 必須 | 最大文字数 | 備考 |
|--------|------|:----:|-----------|------|
| メールアドレス | テキスト入力 | ✓ | 255文字 | `type="email"` |
| パスワード | パスワード入力 | ✓ | - | `type="password"` |

#### ボタン

| ボタン名 | 動作 |
|----------|------|
| ログイン | 認証処理を実行 |

#### バリデーション

| タイミング | 対象 | ルール | エラーメッセージ |
|-----------|------|--------|----------------|
| 送信時 | メールアドレス | 未入力でないこと | 「メールアドレスを入力してください」 |
| 送信時 | メールアドレス | メール形式であること | 「メールアドレスの形式が正しくありません」 |
| 送信時 | パスワード | 未入力でないこと | 「パスワードを入力してください」 |
| 送信時 | 認証 | メール・パスワードが一致すること | 「メールアドレスまたはパスワードが正しくありません」 |

> 認証失敗時のエラーはセキュリティのため「どちらが間違っているか」を明示しない。

#### 操作・遷移

| 操作 | 遷移先 |
|------|--------|
| ログイン成功 | 日報一覧画面 `/reports` |
| ログイン失敗 | 同画面（エラーメッセージ表示） |

---

### 2. 日報一覧画面

**URL:** `/reports`
**アクセス権限:** 全員（要ログイン）

#### 表示項目（一覧テーブル）

| 項目名 | 備考 |
|--------|------|
| 日付 | `report_date`（例: 2026/03/19） |
| 担当者名 | `users.name` |
| Problem（先頭50文字） | 省略表示。未入力は「-」 |
| コメント数 | コメントが0件の場合は「-」 |

#### フィルタ・ソート

| 機能 | 種別 | 備考 |
|------|------|------|
| 開始日 | 日付入力 | 任意。指定日以降の日報に絞り込み |
| 終了日 | 日付入力 | 任意。指定日以前の日報に絞り込み |
| 担当者 | セレクトボックス | **上長のみ表示**。全担当者がデフォルト |
| ソート | - | 日付降順固定 |

#### バリデーション（フィルタ）

| 対象 | ルール | エラーメッセージ |
|------|--------|----------------|
| 開始日・終了日 | 開始日 ≦ 終了日であること | 「開始日は終了日以前の日付を入力してください」 |

#### ボタン・リンク

| 要素 | 表示条件 | 動作 |
|------|---------|------|
| 本日の日報を作成ボタン | 営業担当者かつ当日の日報が未作成 | 日報作成画面へ遷移 |
| 各行（クリック） | - | 日報詳細画面へ遷移 |

#### 操作・遷移

| 操作 | 遷移先 |
|------|--------|
| 本日の日報を作成ボタン | `/reports/new` |
| 行をクリック | `/reports/:id` |

---

### 3. 日報作成画面

**URL:** `/reports/new`
**アクセス権限:** 営業担当者のみ

> 上長がアクセスした場合は `/reports` へリダイレクト。
> 当日の日報が既に存在する場合も `/reports` へリダイレクト。

#### 入力項目

| 項目名 | 種別 | 必須 | 備考 |
|--------|------|:----:|------|
| 日付 | テキスト表示（変更不可） | - | 本日の日付を自動セット |
| 訪問記録 | 複数行入力エリア | 任意 | 後述 |
| Problem（今日の課題・相談） | テキストエリア | 任意 | |
| Plan（明日やること） | テキストエリア | 任意 | |

#### 訪問記録入力エリア

初期状態は0行。「訪問先を追加」ボタンで行を増やす。

| 項目名 | 種別 | 必須 | 備考 |
|--------|------|:----:|------|
| 顧客 | セレクトボックス | ✓（行が存在する場合） | 顧客マスタから選択。「顧客を選択してください」をデフォルト |
| 訪問内容 | テキストエリア | ✓（行が存在する場合） | 自由記述 |
| 行削除ボタン（✕） | ボタン | - | 各行の右端に設置。クリックで行を削除 |
| 訪問先を追加ボタン | ボタン | - | 行を1件追加 |

#### ボタン

| ボタン名 | 動作 |
|----------|------|
| 保存 | 入力内容をバリデーション後、日報を保存 |
| キャンセル | 入力内容を破棄して一覧へ戻る |

#### バリデーション

| タイミング | 対象 | ルール | エラーメッセージ |
|-----------|------|--------|----------------|
| 送信時 | 訪問記録（行が存在する場合）/ 顧客 | 選択されていること | 「顧客を選択してください」 |
| 送信時 | 訪問記録（行が存在する場合）/ 訪問内容 | 未入力でないこと | 「訪問内容を入力してください」 |
| 送信時 | 訪問記録（行が存在する場合）/ 訪問内容 | 1000文字以内であること | 「訪問内容は1000文字以内で入力してください」 |
| 送信時 | Problem | 2000文字以内であること | 「Problemは2000文字以内で入力してください」 |
| 送信時 | Plan | 2000文字以内であること | 「Planは2000文字以内で入力してください」 |

#### 操作・遷移

| 操作 | 遷移先 |
|------|--------|
| 保存成功 | 日報詳細画面 `/reports/:id`（作成された日報） |
| キャンセル | 日報一覧画面 `/reports` |

### 4. 日報詳細画面

**URL:** `/reports/:id`
**アクセス権限:** 全員（要ログイン）

> 存在しないIDの場合は404ページを表示。

#### 表示項目

| 項目名 | 備考 |
|--------|------|
| 日付 | 例: 2026年03月19日（水） |
| 担当者名 | |
| 訪問記録一覧 | 顧客名・訪問内容のペアを登録順に表示。0件の場合は「訪問記録はありません」 |
| Problem | 未入力の場合は「-」 |
| Plan | 未入力の場合は「-」 |
| コメント一覧 | コメント者名・投稿日時・本文を投稿順に表示。0件の場合は「コメントはありません」 |
| コメント投稿フォーム | **上長のみ表示** |
| 編集ボタン | **営業担当者かつ自分の日報のみ表示** |
| 一覧へ戻るリンク | `/reports` へ遷移 |

#### コメント投稿フォーム（上長のみ）

| 項目名 | 種別 | 必須 | 備考 |
|--------|------|:----:|------|
| コメント本文 | テキストエリア | ✓ | |

#### バリデーション（コメント投稿）

| タイミング | 対象 | ルール | エラーメッセージ |
|-----------|------|--------|----------------|
| 送信時 | コメント本文 | 未入力でないこと | 「コメントを入力してください」 |
| 送信時 | コメント本文 | 1000文字以内であること | 「コメントは1000文字以内で入力してください」 |

#### ボタン

| ボタン名 | 表示条件 | 動作 |
|----------|---------|------|
| コメントを送信 | 上長のみ | コメントを保存し同画面を更新 |
| 編集 | 営業担当者かつ自分の日報 | 日報編集画面へ遷移 |

#### 操作・遷移

| 操作 | 遷移先 |
|------|--------|
| 編集ボタン | `/reports/:id/edit` |
| コメント送信 | 同画面（コメント一覧を再表示） |
| 一覧へ戻る | `/reports` |

---

### 5. 日報編集画面

**URL:** `/reports/:id/edit`
**アクセス権限:** 営業担当者（自分の日報のみ）

> 他人の日報IDでアクセスした場合は403ページを表示。
> 上長がアクセスした場合は403ページを表示。
> 存在しないIDの場合は404ページを表示。

#### 入力項目

日報作成画面と同じ構成。既存データを初期値としてセットした状態で表示する。

| 項目名 | 種別 | 必須 | 備考 |
|--------|------|:----:|------|
| 日付 | テキスト表示（変更不可） | - | 保存済みの日付を表示 |
| 訪問記録 | 複数行入力エリア | 任意 | 保存済みの訪問記録を初期表示 |
| Problem | テキストエリア | 任意 | 保存済みの値を初期表示 |
| Plan | テキストエリア | 任意 | 保存済みの値を初期表示 |

#### バリデーション

日報作成画面と同じルールを適用。

#### ボタン

| ボタン名 | 動作 |
|----------|------|
| 保存 | 入力内容をバリデーション後、日報を更新 |
| キャンセル | 入力内容を破棄して詳細画面へ戻る |

#### 操作・遷移

| 操作 | 遷移先 |
|------|--------|
| 保存成功 | 日報詳細画面 `/reports/:id` |
| キャンセル | 日報詳細画面 `/reports/:id` |
